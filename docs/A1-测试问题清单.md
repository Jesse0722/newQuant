# A1：实际走一遍流程 — 问题清单

> 基于 A1 测试清单执行后的记录。已修复项已标注。

---

## 已修复

| 页面 | 问题 | 严重程度 | 修复说明 |
|------|------|----------|----------|
| 仪表盘 | 无加载状态，数据请求期间空白 | 中 | 增加 Spin 加载态 |
| 仪表盘 | 「最新提醒」表格行不可点击 | 低 | 行点击：有 plan_id 跳转计划详情，否则跳转提醒列表 |
| 交易计划详情 | 交易明细「成交日期」为纯文本 Input | 中 | 改为 DatePicker，格式 YYYY-MM-DD，提交时转为 YYYYMMDD |
| 交易计划详情 | 编辑明细时日期为 Input | 中 | 同上，改为 DatePicker |
| 后端 | 仪表盘 recent_alerts 缺少 plan_id | 低 | 补充 plan_id 以支持行点击跳转 |

## 待优化（可选）

| 页面 | 问题 | 严重程度 | 建议 |
|------|------|----------|------|
| 监控提醒 | 无「创建/编辑监控规则」入口 | 中 | 规则创建需从池或股票进入，当前前端未实现该入口，API 已就绪 |
| 交易计划详情 | 成交时间仍为 Input | 低 | 可改为 TimePicker |
| 观察池 | 备注 Input 使用 defaultValue，刷新后可能不同步 | 低 | 可改为受控或加 key 强制重挂载 |

## 验证通过

- 仪表盘：数据展示、卡片点击、空状态
- 观察池：Tab、添加/删除、内联编辑、置顶、创建计划、同步
- 股票详情：K 线、指标切换、关联数据
- 监控提醒：列表、筛选、手动扫描、创建计划/忽略
- 交易计划：列表筛选、新建/编辑、明细、复盘
- 侧边栏导航
