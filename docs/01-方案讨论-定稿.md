# 量化交易系统 - 脑暴记录（2026-03）

> **已定稿**：本次脑暴结论已全部合并进 `01-产品设计-定稿.md`（2026-03 版）。本文件记录完整的讨论过程，供回溯参考。

---

## 脑暴背景

原产品设计（2025-02 版）以「选股 → 回测」闭环为核心。经过本次重新讨论，产品方向做了重大调整：

- **旧方向**：选股 → 股票池管理 → 回测验证
- **新方向**：选股 → 监控提醒 → 交易计划 → 执行记录 → 复盘

调整原因：回测平台市面上已经很多，无需重复造轮子。真正的痛点是**交易工作流管理**——搞清楚为什么交易、记录交易、复盘总结。

---

## 一、产品定位重新梳理

### 讨论

**问**：核心使用场景是什么？

**答**：两个场景——
1. 使用灵活的策略进行选股，加入观察池，观察池有监控功能，对进入买点的股票提醒
2. 交易计划管理：选股 → 制定次日交易计划 → 记录交易 → 复盘。重点在于每次交易要搞清楚「为什么交易」和「结果是否符合预期」

### 结论

产品是一个**交易工作流管理工具**，而非回测平台。

---

## 二、交易计划模块

### 讨论

**问**：交易计划是单股还是多股？

**答**：单股。每个交易计划对应一只股票，通过标签/类型分组关联。

**问**：交易计划字段？

**答**：确认以下字段——

**已确认字段：**
- 标的（股票代码 + 名称）
- 计划类型（趋势跟踪 / 短线操作 / 事件驱动）
- 触发策略（关联选股记录）
- 热点/事件（宏观/事件背景）
- 计划买入价、目标价、止损价
- 盈亏比（自动计算）
- 计划仓位（金额或比例）
- 操作建议（买入 / 加仓 / 观望）
- 风险等级（1-5 星）
- 计划状态（待触发 / 执行中 / 已完结 / 已取消）
- 备注

**去掉的字段：**
- ~~有效期~~ — 时间不固定，得根据股票走势决定

**复盘放在计划级别（非交易明细级别）：**
- 实际盈亏（从交易明细自动汇总）
- 复盘总结（整体执行评价）
- 经验教训

---

## 三、交易明细模块

### 讨论

**问**：交易明细字段？

**答**：确认以下字段——
- 成交日期、成交时间（可选）
- 方向（买入 / 卖出）
- 成交价格、成交数量、成交金额（自动计算）
- 佣金（可选，按比例自动算）、印花税（卖出时自动算）
- 执行备注（当时的判断依据）

交易明细手动添加，关联到交易计划。复盘评价不在明细层面，统一放在计划级别。

---

## 四、整体工作流确认

### 讨论

**问**：模块串联关系是否正确？

核心链路：数据同步 → 观察池 → 监控扫描/买点提醒 → 创建交易计划 → 手动记录交易 → 复盘

**答**：正确。观察池也可以直接手动创建交易计划（不一定等买点提醒）。

---

## 五、MVP 范围

### 讨论

**第一轮**：用户提出数据同步和策略选股 V1 都不做，V1 只做观察池 + 交易计划。

**第二轮**：用户意识到监控是核心功能，监控需要行情数据，所以数据同步必须做。但策略选股仍不做，V1 所有股票由人工导入。

**最终结论**：

**V1 包含：**
1. 观察池（多分组、单个添加 + CSV 批量导入）
2. 数据同步（仅观察池中股票，基础信息 + 日线行情）
3. 监控提醒（6 模板 + 自定义，系统内提醒）
4. 交易计划 + 交易明细 + 复盘

**V2 包含：**
- AI 工作流驱动选股（一键导入观察池）
- 回测模块
- 微信/钉钉推送
- 周期性复盘报告、PDF 导出
- 实盘接口

## 六、观察池模块

### 讨论

**问**：多分组怎么理解？

**答**：扁平分组。多个独立的观察池（如「短线池」「趋势池」），每个池子独立管理。不用标签分组。

**问**：池内股票字段？

**答**：确认——股票代码、名称（自动）、加入日期（自动）、**加入价格**、加入来源、监控状态、监控条件（继承池子默认/可覆盖）、备注。

**问**：池子本身字段？

**答**：确认——名称、描述（可选）、默认监控条件（新股票自动继承）、股票数量（自动）、创建时间。

**问**：CSV 导入格式？

**答**：灵活匹配。必填仅股票代码，其他列（价格、备注等）有就读没有就跳过。

---

## 七、监控提醒模块

### 讨论

**问**：买点条件类型？

**答**：模板 + 自定义。提供常用模板，也支持自定义组合（AND/OR）。

**问**：预置模板？

**答**：6 个全部保留——
1. 均线支撑（收盘价回踩 MA(N)）
2. MACD 金叉（DIF 上穿 DEA）
3. RSI 超卖（RSI < 阈值）
4. 缩量回调（价格回调 + 量缩）
5. 突破新高（收盘价创 N 日新高）
6. 价格阈值（跌破指定价格，如加入价 × 95%）

**问**：扫描触发时机？

**答**：数据同步后自动扫描 + 随时手动触发，两者都支持。

**问**：提醒方式？

**答**：V1 系统内消息列表/弹窗；V2 加微信/钉钉 webhook。

## 八、数据同步模块

### 讨论

**问**：数据源？

**答**：继续用 Tushare Pro，观察池规模不大，够用。

**问**：同步范围和策略？

**答**：
- 仅同步观察池中的股票（不做全市场）
- 股票基础信息 + 日线行情
- 增量同步（只拉缺失日期）
- 加入观察池时自动拉取近 **250 天**历史数据
- 手动同步
- 同步完成后自动触发监控扫描

---

## 九、页面结构

### 讨论

**问**：侧边栏怎么分？

**答**：4 块——仪表盘、观察池、监控提醒、交易计划。

**问**：复盘需要单独页面吗？

**答**：不需要。复盘放在交易计划详情页里。

**最终路由：**
- `/` — 仪表盘
- `/pools` — 观察池列表
- `/pools/:id` — 池内详情
- `/alerts` — 监控提醒
- `/plans` — 交易计划列表
- `/plans/:id` — 计划详情（含交易明细 + 复盘）

## 十、技术架构

### 讨论

**问**：技术栈是否调整？

**答**：保持不变——React + TypeScript + Ant Design + Vite（前端）、Python + FastAPI（后端）、SQLite（存储）、Tushare Pro（数据源）、后台线程（异步任务）。

## 十一、与旧版设计的主要差异

| 维度 | 旧版（2025-02） | 新版（2026-03） |
|------|-----------------|-----------------|
| 核心定位 | 策略研发 + 回测验证 | 交易工作流管理 |
| 回测 | MVP 核心模块 | V2，非核心 |
| 选股 | 因子 + 条件筛选 | V1 人工导入，V2 AI 选股 |
| 股票池 | 静态管理 + 快照 | 观察池 + 监控提醒（动态） |
| 交易计划 | 无 | 核心模块（计划 + 明细 + 复盘） |
| 监控 | 无 | 核心模块（6 模板 + 自定义 + 自动扫描） |
| 数据同步 | 全市场 | 仅观察池中股票 |
